# CH Router Visualization

Interactive D3.js visualization of the Contraction Hierarchies (CH) routing algorithm vs Dijkstra on the Berlin road network.

Created by Steffen Axer ([iteratively.io](https://iteratively.io)) to demonstrate CH routing efficiency for LinkedIn.

## Files

| File | Description |
|---|---|
| `ch-visualization.html` | Standalone interactive visualization (open in browser) |
| `ch-query-data.json` | Pre-computed query data (generated by `CHQueryExporter`) |
| `plan-chRouterVisualization.prompt.md` | Implementation plan |

## Quick Start

### Step 1: Generate the query data

Run the `CHQueryExporter` from the `matsim-libs` root directory:

```bash
cd matsim-libs

# Install parent + examples first (if not done)
mvn install -N -DskipTests
mvn install -pl examples -DskipTests

# Run the exporter
mvn exec:java -pl matsim \
  -Dexec.mainClass=org.matsim.core.router.speedy.CHQueryExporter \
  -Dexec.args="examples/scenarios/berlin/network.xml.gz matsim/tools/ch-viz/ch-query-data.json"
```

This takes 2–5 minutes (CH preprocessing for the Berlin network).
Output: `matsim/tools/ch-viz/ch-query-data.json` (~3–5 MB)

### Step 2: Open the visualization

**Option A — Local web server (recommended):**
```bash
cd matsim/tools/ch-viz
python3 -m http.server 8080
# Open http://localhost:8080/ch-visualization.html
```

**Option B — Direct file open:**
Most browsers block `fetch()` from `file://` URLs. Use a local server as above, or use Firefox (which allows local file fetching).

## Visualization Features

- **Split-screen** Dijkstra vs CH comparison (synchronized animation)
- **Play / Pause / Step** controls with speed slider (1×–50× events per frame)
- **Progress bar** with click-to-seek
- **2.5D mode** — nodes lifted by their CH contraction level, showing the hierarchy
- **Meeting point** golden glow when the bidirectional CH search converges
- **Stall-on-demand** visualization (gray flash for pruned nodes)
- **Final overlay** showing search space savings (e.g. "97% less search space")
- **Cinema mode** — hides controls, auto-plays for screen recording
- **Video export** — native `canvas.captureStream(60fps)` + `MediaRecorder` → downloads `ch-router-berlin.webm`
- **4 pre-computed OD pairs** selectable via dropdown
